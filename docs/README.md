## Guides

Guides are provided by the ArcKit plugin. Run any `/arckit.*` command and refer to the plugin documentation for detailed usage instructions.

### Core Workflow
1. `/arckit.plan` - Project Plan
2. `/arckit.principles` - Architecture Principles
3. `/arckit.stakeholders` - Stakeholder Analysis
4. `/arckit.risk` - Risk Management
5. `/arckit.sobc` - Business Case
6. `/arckit.requirements` - Requirements Definition

### Research & Analysis
- `/arckit.research` - Market Research
- `/arckit.datascout` - Data Source Discovery
- `/arckit.aws-research` - AWS Service Research
- `/arckit.azure-research` - Azure Service Research
- `/arckit.gcp-research` - GCP Service Research

### Design & Implementation
- `/arckit.data-model` - Data Model
- `/arckit.dpia` - Data Protection Impact Assessment
- `/arckit.wardley` - Wardley Mapping
- `/arckit.adr` - Architecture Decision Records
- `/arckit.diagram` - Architecture Diagrams

### Operations
- `/arckit.devops` - DevOps Strategy
- `/arckit.mlops` - MLOps Strategy
- `/arckit.finops` - FinOps Strategy
- `/arckit.operationalize` - Operational Readiness

### Compliance
- `/arckit.secure` - Secure by Design
- `/arckit.tcop` - Technology Code of Practice
- `/arckit.ai-playbook` - AI Playbook Assessment
- `/arckit.atrs` - Algorithmic Transparency
